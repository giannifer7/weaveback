= AsciiDoc AST Exploration
:toc: left
:source-highlighter: syntect

This example demonstrates how Weaveback might utilize the `asciidoc-parser` crate to construct an internal semantic AST replacing the current Regex-based and Ruby parsing mechanisms.

== Configuration

We configure a standard Rust binary project for this exploration. It relies directly on `asciidoc-parser` from its GitHub source, matching Weaveback's workspace `edition = "2024"`.

[source,toml]
----
# <[@file Cargo.toml]>=
[package]
name = "ast_parser"
version = "0.1.0"
edition = "2024"

[dependencies]
asciidoc-parser = { git = "https://github.com/asciidoc-rs/asciidoc-parser" }
# @@
----

== Implementation

The AST parser is remarkably straightforward. We instantiate `asciidoc_parser::Parser::default()`, pull the file content dynamically via `fs::read_to_string`, and dump the resulting AST using `{:#?}` to inspect how well it structures Weaveback syntax blocks.

[source,rust]
----
// <[@file src/main.rs]>=
use asciidoc_parser::Parser;
use std::env;
use std::fs;

fn main() {
    let args: Vec<String> = env::args().collect();
    if args.len() < 2 {
        eprintln!("Usage: {} <file.adoc>", args[0]);
        std::process::exit(1);
    }

    let file_path = &args[1];
    let source = match fs::read_to_string(file_path) {
        Ok(content) => content,
        Err(e) => {
            eprintln!("Failed to read file {}: {}", file_path, e);
            std::process::exit(1);
        }
    };

    let mut parser = Parser::default();
    
    // Parse the UTF-8 source text
    let document = parser.parse(&source);
    
    println!("{:#?}", document);
}
// @@
----

== Automation

We include a simple `justfile` to orchestrate Weaveback tangling and executing the compiled `cargo run`. Note that we specify `--chunk-end @@` to match our marker style.

[source,just]
----
# <[@file justfile]>=
tangle:
    ../../target/debug/weaveback ast_parser.adoc --gen . --db ast_parser.db --chunk-end @@

run: tangle
    cargo run --release -- ../../crates/weaveback/src/lib.adoc
# @@
----
